EduPlatform  Платформа  для  онлайн-курсов  
   
Описание  
Необходимо  разработать  backend- систему  для  платформы  онлайн-обучения,  состоящую  
из
 
двух
 
сервисов
.
 
Платформа
 
позволяет

создавать

с

а

Архитектура:  два  сервиса  Сервис  1  —  Main  API  (Nest.js)  
Основной  сервер,  отвечающий  за  всю  бизнес-логику:    
●  Авторизация  и  регистрация  с  использованием  JWT- токенов  ●  Две  роли :  student  и  teacher  ●  CRUD  курсов  (создание,  просмотр,  редактирование,  удаление)  —  доступно  только  
преподавателям-владельцам
 ●  CRUD  уроков  внутри  курсов  ●  Запись  студента  на  курс  ●  Загрузка  изображений  (обложки  курсов  и  картинки  к  урокам)  с  последующей  
отправкой
 
события
 
в
 
Kafka
 ●  Кэширование  списка  курсов  и  детальной  информации  о  курсе  через  Redis  с  
инвалидацией
 
кэша
 
при
 
изменении
 
данных
   
Сервис  2  —  Image  Worker  (Express.js)  
Отдельный  микросервис  для  обработки  изображений:    
●  Слушает  топик  Kafka  и  получает  задачи  на  обработку  ●  Сжимает  изображение  до  разумного  размера  и  качества  (библиотека  Sharp)  
--- PAGE 2 ---
●  Накладывает  водяной  знак  (watermark)  ●  Сохраняет  обработанный  файл  ●  Обновляет  статус  изображения  в  базе  данных  (например,  processing  →  ready )  ●  После  обработки  отправляет  событие  в  Kafka  о  завершении    
   
Технологический  стек  
  
Технология  Где  и  зачем  используется  Nest.js  Main  API  —  основная  бизнес-логика  Express.js  Image  Worker  —  микросервис  обработки  изображений  MongoDB  Основная  база  данных  Mongoose  ODM  для  работы  с  MongoDB  (схемы,  валидация,  связи)  Redis  Кэширование  данных  о  курсах  с  инвалидацией  Kafka  Асинхронное  взаимодействие  между  сервисами  (два  топика:  отправка  задачи  на  обработку  и  уведомление  о  завершении)  Sharp  Сжатие  изображений  и  наложение  водяного  знака

Требуемые  API- эндпоинты  Авторизация  
●  Регистрация  (с  указанием  роли)  ●  Логин  (возвращает  JWT- токен)    
Курсы  
●  Получить  список  всех  курсов  (кэшируется  в  Redis)  ●  Получить  детали  курса  по  ID  (кэшируется  в  Redis)  ●  Создать  курс  (только  teacher)  ●  Обновить  курс  (только  teacher- владелец,  инвалидирует  кэш)  ●  Удалить  курс  (только  teacher- владелец,  инвалидирует  кэш)  ●  Записаться  на  курс  (только  student)    
Уроки  
●  Получить  все  уроки  курса  ●  Добавить  урок  (только  teacher- владелец  курса)  ●  Обновить  урок  ●  Удалить  урок    
Загрузка  изображений  
●  Загрузить  обложку  курса  ●  Загрузить  изображение  к  уроку  ●  Получить  обработанное  изображение  по  имени  файла    
   
Поток  обработки  изображений  
--- PAGE 4 ---
1.  Преподаватель  загружает  картинку  через  Main  API  2.  Файл  сохраняется  в  папку  с  оригиналами,  в  базе  статус  —  processing  3.  Main  API  отправляет  сообщение  в  Kafka  (топик  image.uploaded )  с  информацией  
о
 
файле
 4.  Image  Worker  получает  сообщение,  обрабатывает  картинку  (сжатие  +  водяной  знак)  5.  Обработанный  файл  сохраняется  в  отдельную  папку  6.  Image  Worker  обновляет  статус  в  MongoDB  на  ready  7.  Image  Worker  отправляет  сообщение  в  Kafka  (топик  image.processed )  —  
подтверждение
